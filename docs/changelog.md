# 更新履歴

## 2026-03-14

### UX改善

#### 未保存変更インジケーター
- 入力を変更すると右下に `● 未保存の変更があります`（オレンジ）を表示
- 保存後は自動的に消える

#### 自動保存（下書き）
- 入力内容をブラウザのローカルストレージに自動保存
- 再訪時にサーバーデータと差異がある場合、`📝 未保存の下書きが見つかりました` バナーを表示
- 「復元する」または「破棄」で選択可能
- 30日以上前の下書きは自動削除

#### 未保存離脱警告
- 未保存の状態でページを閉じる・別ページに移動しようとするとブラウザの確認ダイアログを表示

#### 夜勤巡視「前と同じ」ボタン
- 各セルのポップオーバーに `← 前と同じ（氏名）` ボタンを追加
- 直前の時間帯に記録がある場合のみ表示

#### 入居者数・避難区分・介護度 整合性チェック
- 1F・2F それぞれでリアルタイムに合計人数を比較
- 一致しないフロアがある場合、赤いバナーで警告表示（印刷には出ない）
- 支援も介護度総数に含めてチェック

#### 平均介護度の計算方法変更
- 変更前：加重合計 ÷ 入居者総数
- 変更後：加重合計 ÷ (入居者総数 − 支援総数)
- 計算式の表示も「÷ ○名（支援除く）」に更新

#### 「前日から取り込む」ボタンの視認性向上
- アクセントカラー（青）・太字・影付きに変更

### パフォーマンス改善

#### Google Sheets 読み込み最適化
- 日報取得を A:B 全行読み込みから2段階に変更
  1. A列（日付のみ）で行番号を特定
  2. 対象行の B列のみ読み込む
- 行数が増えても転送データ量がほぼ一定に
- 保存時の Sheets 読み込みを 2回 → 1回に削減

#### 保存後の `loadedAt` 正確化
- 保存 API がサーバー設定の `updatedAt` をレスポンスに含めて返すよう変更
- 同時編集の競合検知（ポーリング）の誤検知を防止

### バグ修正

- 他PCで起動した際に今日の日付ではなくサーバーの日付が表示される問題を修正（クライアントサイドで日付を計算するよう変更）
